- (decision) Adopt Next.js App Router + TypeScript with client-first hexagonal architecture; binds public SEO pages, converter workflow, server OCR, exports, ads/analytics boundaries, and privacy receipts; prevents a pure SPA weakening SEO or a backend-first design violating the local-processing privacy promise.

- (decision) Route processing by detected document mode: digital PDFs use local browser extraction first, scanned/image PDFs require explicit consent before server OCR, and password handling stays local until server processing is explicitly approved; binds FR-5 through FR-8 and FR-13; prevents silent upload of sensitive PDFs or passwords.
- (decision) Isolate server OCR as a temporary job boundary with opaque job ids, short-lived object storage, artifact TTL under 30 minutes, and cleanup status surfaced to the receipt path; binds OCR server processing, exports generated server-side if any, observability, and cleanup copy; prevents long-lived document storage and false deletion claims.
- (decision) Keep ads and analytics in public/content shells only, with a no-sensitive-data telemetry contract for converter events; binds FR-3, privacy guardrails, SEO pages, metrics, and counter-metrics; prevents ad or analytics adapters from observing files, extracted rows, passwords, or preview edits.
- (decision) Generate Excel/CSV exports from the current preview model, not directly from original extraction output; binds FR-10 through FR-12; prevents user edits from being lost or export columns drifting from the reviewed preview.
- (decision) Use source-derived table columns as the canonical preview/export schema per conversion, with optional internal metadata for confidence and mapping; binds FR-9, TEST-3, and exports; prevents forcing every statement into one global bank-transaction schema.
- (decision) Use static/server-rendered SEO route content that links into the shared converter shell; binds SEO-1 through SEO-5; prevents thin duplicated pages and JavaScript-only acquisition pages.
- (decision) Enforce intake and resource safeguards on every file path, including local detection limits and server upload/OCR rate, size, type, timeout, and malformed-PDF defenses; binds FR-4, security NFR, capacity/abuse protection, TEST-4; prevents the no-artificial-product-limit promise from disabling operational safeguards.
